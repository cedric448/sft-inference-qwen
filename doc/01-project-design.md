# 项目设计文档

## 目标

对 Qwen3.5-0.8B 基础大语言模型进行中文通用法律领域的监督微调（SFT），使其具备专业的法律知识问答能力，输出风格简洁、准确。

## 技术选型

| 维度 | 选择 | 理由 |
|------|------|------|
| 基础模型 | Qwen3.5-0.8B | 参数量小（0.8B），推理快，适合单卡部署；支持中文 |
| 微调方式 | LoRA（rank=16, alpha=32） | 显存占用低（<15GB），训练快，适合法律知识注入 |
| 训练框架 | LLaMA-Factory 0.9.5 | 支持 Qwen3 template，配置简洁，社区活跃 |
| 数据集 | DISC-Law-SFT | 中文法律 SFT 数据集，覆盖多类型法律问答，质量高 |
| 推理服务 | FastAPI + transformers | Qwen3.5 架构较新，vLLM 尚不完整支持，直接推理兼容性更好 |
| 硬件 | NVIDIA L40 48GB | 单卡即可完成 LoRA 训练和推理 |

## 数据策略

- **来源**：DISC-Law-SFT（AI-ModelScope/DISC-Law-SFT）
  - Pair 子集：143,374 条（input/output 格式）
  - Triplet 子集：16,000 条（含参考文献）
- **采样**：从 159,374 条中随机采样 8,000 条（seed=42）
- **格式转换**：转为 ShareGPT JSON 格式，system prompt 统一注入
- **system prompt**：`你是一位专业的中国法律顾问，请根据用户的问题提供准确、简洁的法律解答。`

## 质量过滤

- input 长度 > 5 字符
- output 长度 > 10 字符
- 过滤空行和无效数据
